# Feature Specification: ページ構造リファクタリング

**Feature Branch**: `006-page-structure-refactor`
**Created**: 2025-12-05
**Status**: Draft
**Input**: User description: "ページの構造をリファクタしてほしいです。セッション管理画面・ログイン画面・トップページというシンプルな構造なのに、ページ間の移動の不便や同じアクションが複数の場所からできるようになっている等UXの低下を招いている点が複数あるので、整理してほしいです。"

## Clarifications

### Session 2025-12-05

- Q: ページ構造のリファクタリングについて、どのアプローチを希望しますか？ → A: 専用ページ中心（モーダルを削除し、専用ページで操作）
- Q: グローバルナビゲーション（ヘッダー/ナビバー）を追加しますか？ → A: サイドバー（折りたたみ可能、Mantine AppShellを使用）
- Q: トップページ（/）の役割をどうしますか？ → A: ダッシュボード（サマリー統計やクイックアクション表示）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - サイドバーナビゲーションによるページ移動 (Priority: P1)

ユーザーは、アプリケーション内のどのページからでもサイドバーを使って主要な機能（ダッシュボード、セッション一覧）へ1クリックで移動できる。デスクトップでは常に展開されたサイドバーが表示され、モバイルデバイスではハンバーガーメニューとして表示される。

**Why this priority**: グローバルナビゲーションがないことが現在の最大のUX問題。すべてのページ改善の基盤となるため最優先。

**Independent Test**: ログイン後、任意のページからサイドバーを使って別のページへ移動し、元のページに戻れることで価値を検証できる。

**Acceptance Scenarios**:

1. **Given** ログイン済みユーザーがダッシュボードにいる、**When** サイドバーの「セッション一覧」をクリックする、**Then** セッション一覧ページに遷移する
2. **Given** ログイン済みユーザーがセッション一覧にいる、**When** サイドバーの「ダッシュボード」をクリックする、**Then** ダッシュボードページに遷移する
3. **Given** モバイルデバイスでアプリを開いている、**When** ハンバーガーメニューをタップする、**Then** サイドバーがオーバーレイで表示される
4. **Given** デスクトップでサイドバーが表示されている、**When** 画面を確認する、**Then** サイドバーは常に展開された状態で表示される

---

### User Story 2 - ダッシュボードでの概要確認 (Priority: P2)

ユーザーは、ログイン後にダッシュボード（トップページ）でセッションのサマリー統計（総利益、セッション数、勝率など）と最近のセッションを確認できる。クイックアクションボタンから主要な操作にすぐアクセスできる。

**Why this priority**: ログイン直後に重要な情報を提供することで、ユーザーの次のアクションを導く。サイドバーの次に重要なページ改善。

**Independent Test**: ログイン後にダッシュボードが表示され、統計情報と最近のセッションが正しく表示されることで価値を検証できる。

**Acceptance Scenarios**:

1. **Given** ログイン済みユーザーがトップページにアクセスする、**When** ページが読み込まれる、**Then** サマリー統計（総利益、セッション数、平均利益/損失）が表示される
2. **Given** 過去にセッションを記録している、**When** ダッシュボードを表示する、**Then** 最近のセッション（最大5件）が一覧表示される
3. **Given** ダッシュボードを表示している、**When** 「新規セッション作成」ボタンをクリックする、**Then** 新規作成ページに遷移する
4. **Given** セッションが0件の状態、**When** ダッシュボードを表示する、**Then** 「最初のセッションを記録しましょう」という案内が表示される

---

### User Story 3 - 専用ページでのセッション操作 (Priority: P3)

ユーザーは、セッションの作成・詳細表示・編集・削除をそれぞれ専用のページで行える。モーダルではなく専用ページを使用することで、URLの共有やブラウザの戻る/進むボタンが正しく機能する。

**Why this priority**: 現在はモーダルと専用ページが混在しており、ユーザー体験が一貫していない。専用ページに統一することで、ナビゲーションの一貫性とURLベースの操作が可能になる。

**Independent Test**: 新規作成、詳細表示、編集のそれぞれがブックマーク可能なURLを持ち、ブラウザの戻るボタンで前のページに戻れることで価値を検証できる。

**Acceptance Scenarios**:

1. **Given** セッション一覧ページにいる、**When** 「新規作成」ボタンをクリックする、**Then** `/poker-sessions/new`に遷移し、作成フォームが表示される
2. **Given** セッション一覧ページでセッションカードをクリックする、**When** ページが遷移する、**Then** `/poker-sessions/[id]`で詳細情報が表示される
3. **Given** セッション詳細ページにいる、**When** 「編集」ボタンをクリックする、**Then** `/poker-sessions/[id]/edit`に遷移し、編集フォームが表示される
4. **Given** 編集フォームで変更を保存する、**When** 保存が完了する、**Then** `/poker-sessions/[id]`にリダイレクトされ、更新された内容が表示される
5. **Given** セッション詳細ページにいる、**When** 「削除」ボタンをクリックして確認する、**Then** セッションが削除され、セッション一覧ページにリダイレクトされる
6. **Given** `/poker-sessions/new`のURLを直接入力する、**When** ページが読み込まれる、**Then** 新規作成フォームが正しく表示される

---

### User Story 4 - 認証ページの独立表示 (Priority: P4)

未認証ユーザーは、サイドバーなしのシンプルなレイアウトでログイン・登録ページにアクセスできる。認証完了後はダッシュボードにリダイレクトされる。

**Why this priority**: 認証ページは独立したフローであり、AppShellレイアウトから除外することで、認証に集中できるシンプルなUIを維持する。

**Independent Test**: 未認証状態でログインページにアクセスし、サイドバーなしの画面が表示されることで価値を検証できる。

**Acceptance Scenarios**:

1. **Given** 未認証ユーザーがログインページにアクセスする、**When** ページが表示される、**Then** サイドバーは表示されない
2. **Given** 未認証ユーザーがログインに成功する、**When** リダイレクトが行われる、**Then** ダッシュボード（/）に遷移し、サイドバーが表示される
3. **Given** 認証済みユーザーがログアウトする、**When** ログアウトが完了する、**Then** ログインページにリダイレクトされ、サイドバーは非表示になる

---

### Edge Cases

- 存在しないセッションIDにアクセスした場合 → 404ページまたはセッション一覧へリダイレクト
- 他ユーザーのセッションにアクセスしようとした場合 → 403エラーまたはセッション一覧へリダイレクト
- モバイルでサイドバーを開いた状態でページ遷移した場合 → サイドバーは自動的に閉じる
- ブラウザの戻るボタンでモーダルから戻ろうとした場合 → モーダル削除により発生しない（専用ページのため正常動作）

## Requirements *(mandatory)*

### Functional Requirements

**サイドバーナビゲーション**:
- **FR-001**: システムは、Mantine AppShellを使用したサイドバーを提供しなければならない（デスクトップでは常に展開表示、モバイルではハンバーガーメニュー）
- **FR-002**: サイドバーは、ダッシュボード、セッション一覧へのリンクを含まなければならない
- **FR-003**: サイドバーは、現在のページを視覚的にハイライト表示しなければならない
- **FR-004**: モバイルデバイス（768px以下）では、サイドバーはハンバーガーメニューとして動作しなければならない
- **FR-005**: サイドバーは、認証済みユーザーにのみ表示されなければならない

**ダッシュボード**:
- **FR-006**: トップページ（/）は、ログイン後にダッシュボードとして機能しなければならない
- **FR-007**: ダッシュボードは、セッション統計（総利益、セッション数、勝率）を表示しなければならない
- **FR-008**: ダッシュボードは、最近のセッション（最大5件）を表示しなければならない
- **FR-009**: ダッシュボードは、新規セッション作成へのクイックアクションボタンを提供しなければならない

**専用ページ**:
- **FR-010**: `/poker-sessions`は、セッション一覧とフィルター機能を提供しなければならない（モーダルなし）
- **FR-011**: `/poker-sessions/new`は、新規セッション作成フォームを提供しなければならない
- **FR-012**: `/poker-sessions/[id]`は、セッション詳細表示ページを提供しなければならない
- **FR-013**: `/poker-sessions/[id]/edit`は、セッション編集フォームを提供しなければならない
- **FR-014**: 編集完了後は`/poker-sessions/[id]`にリダイレクトしなければならない
- **FR-015**: 削除完了後は`/poker-sessions`にリダイレクトしなければならない

**認証ページ**:
- **FR-016**: `/auth/signin`と`/auth/signup`は、サイドバーなしで表示されなければならない
- **FR-017**: 認証成功後は、ダッシュボード（/）にリダイレクトしなければならない

**モーダル削除**:
- **FR-018**: セッション作成・編集用のモーダルコンポーネントを削除しなければならない
- **FR-019**: 削除確認ダイアログのみモーダル形式を維持してもよい

### Key Entities

- **AppLayout**: Mantine AppShellを使用したレイアウトコンポーネント。サイドバーの表示/非表示、折りたたみ状態を管理
- **Navbar**: サイドバー内のナビゲーションリンクとユーザーメニューを含むコンポーネント
- **SessionDetail**: セッション詳細表示用のPresentationコンポーネント

## Success Criteria *(mandatory)*

### Measurable Outcomes

**ナビゲーション効率**:
- **SC-001**: ユーザーは任意のページから他の主要ページへ2クリック以内で移動できる
- **SC-002**: モバイルユーザーは3タップ以内で新規セッションを作成開始できる

**ページ構造の一貫性**:
- **SC-003**: すべてのセッション操作（作成・詳細・編集）がブックマーク可能なURLを持つ
- **SC-004**: ブラウザの戻る/進むボタンが全ページで正常に動作する
- **SC-005**: 同じ操作が複数の場所から行える冗長なUIが存在しない

**レスポンシブ対応**:
- **SC-006**: 375px以上の画面幅で全ページが正常に表示・操作可能である
- **SC-007**: デスクトップとモバイルで一貫したナビゲーション体験を提供する

**パフォーマンス**:
- **SC-008**: ダッシュボードの初期表示が2秒以内に完了する
- **SC-009**: ページ間遷移が500ms以内に完了する
