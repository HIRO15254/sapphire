mainブランチに向けて現在出されているPRの内容からアップデートノートを作成し、バージョンを上げてコミットしてください。

## 手順

1. `gh pr list --base main --state open` で現在のPRを特定する
2. `gh pr view <number>` でPRの全コミット情報を取得する
3. `git log main..HEAD --oneline` でコミット一覧を確認する
4. 既存の `CHANGELOG.md` のフォーマットに従い、コミット内容を分類してアップデートノートを作成する
   - **Added**: 新機能
   - **Changed**: 既存機能の変更・リファクタリング
   - **Fixed**: バグ修正
   - chore や CI のみの変更はユーザー向けでないため省略する
5. `package.json` の `version` をセマンティックバージョニングに従いバンプする
   - 破壊的変更がある場合: メジャー
   - 新機能追加がある場合: マイナー
   - バグ修正のみの場合: パッチ
6. 変更を確認してコミットする（メッセージ: `release: v<新バージョン>`）
