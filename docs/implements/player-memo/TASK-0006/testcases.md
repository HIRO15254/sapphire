# TASK-0006: 種別影響カウントコマンド実装 - TDDテストケース定義書

## 🎯 テスト対象機能

**機能名**: `get_affected_players_count_by_category`

**コマンド**:
```rust
#[tauri::command]
pub async fn get_affected_players_count_by_category(
    category_id: i64,
    db: State<'_, PlayerDatabase>,
) -> Result<usize, String>
```

**目的**: プレイヤー種別を編集・削除する前に、その種別が割り当てられているプレイヤーの数をカウントして返す

  - **システムの安全性**: 不正なIDがSQLインジェクションの原因にならない
- **テストの目的**: 入力値バリデーションが正しく機能することを確認
  - **品質保証の観点**:
    - パラメータバインディングによるSQLインジェクション対策

#### TC-COUNT-CATEGORY-BOUND-003: category_idがNULLのプレイヤーは含まれない 🔵

  - **境界値の意味**: `category_id = NULL`のプレイヤーは種別未割り当て（どの種別にも属さない）
  - **境界値での動作保証**: WHERE句が `category_id = ?` のため、NULL値は含まれない

- **期待される結果**: `Ok(1)`（category_id = 1のプレイヤーAのみカウント）
  - **境界での正確性**: NULLは`WHERE category_id = 1`にマッチしない
  - **一貫した動作**: SQL仕様通りNULL値を除外
- **テストの目的**: NULL値のハンドリングが正しいことを確認
  - **堅牢性の確認**:
    - NULL値が誤ってカウントされない
    - ON DELETE SET NULLとの整合性
    - SQL仕様に準拠した動作
- **信頼性レベル**: 🔵 データベース制約（ON DELETE SET NULL、schema.rs:18）に基づく
