# TDD要件定義・機能仕様の整理

**【機能名】**: ResponsiveLayout（最上位）コンポーネント

## 1. 機能の概要（EARS要件定義書・設計文書ベース）

- 🟢 **主要機能**: デバイスの画面サイズ（768px ブレークポイント）に基づいて自動的にモバイル・デスクトップレイアウトを切り替える最上位レイアウトコンポーネント
- 🟢 **問題解決**: スマートフォンユーザーとデスクトップユーザーそれぞれに最適化されたナビゲーション構造を提供し、効率的なアプリ操作を実現
- 🟢 **想定ユーザー**:
  - スマートフォンユーザー（フッターメニュー + ハンバーガーメニューでの操作）
  - デスクトップユーザー（ヘッダーメニュー + サイドメニューでの操作）
- 🟢 **システム内位置づけ**: Component-Based Architectureの最上位レイアウトコンポーネントとして、MantineProvider + AppShell統合により全アプリケーションのレイアウト基盤を提供
- **参照したEARS要件**: REQ-001, REQ-002, REQ-101, REQ-102
- **参照した設計文書**: architecture.md「アーキテクチャ層構成」「主要コンポーネント」セクション

## 2. 入力・出力の仕様（EARS機能要件・TypeScript型定義ベース）

### 入力パラメータ
- 🟢 **children**: `React.ReactNode` - メインコンテンツエリアに表示する子コンポーネント
- 🟢 **navigationConfig**: `NavigationConfig` - ナビゲーション項目の設定オブジェクト
  - `primary: NavigationItem[]` - 主要ナビゲーション項目（ヘッダー・フッター用）
  - `secondary: NavigationItem[]` - 補助ナビゲーション項目（サイド・ハンバーガー用）
- 🟡 **theme**: `'light' | 'dark' | 'auto'` (オプション、デフォルト: 'auto') - テーマ設定
- 🟡 **primaryColor**: `string` (オプション、デフォルト: 'blue') - プライマリカラー設定

### 出力値
- 🟢 **JSX要素**: 画面サイズに応じて適切なレイアウトで構成されたアプリケーション全体
- 🟢 **モバイルレイアウト** (768px以下):
  - Header: ハンバーガーメニューボタン + ロゴ + テーマ切り替え
  - Footer: タブバー形式のメインナビゲーション
  - Drawer: ハンバーガーメニュー（サブナビゲーション）
- 🟢 **デスクトップレイアウト** (769px以上):
  - Header: ロゴ + 水平メインナビゲーション + テーマ切り替え
  - Sidebar: 折りたたみ可能なサブナビゲーション

### データフロー
- 🟢 **useMediaQuery('(max-width: 48em)')** → **isMobile状態** → **レイアウト条件分岐**
- 🟢 **NavigationProvider** → **ナビゲーション状態管理** → **各ナビゲーションコンポーネント**

- **参照したEARS要件**: REQ-101, REQ-102
- **参照した設計文書**: mantine-component-design.md「ResponsiveLayoutProps インターフェース」

## 3. 制約条件（EARS非機能要件・アーキテクチャ設計ベース）

### パフォーマンス要件
- 🟢 **NFR-001**: レイアウト切り替えは200ms以内で完了すること
- 🟢 **NFR-002**: メニューの開閉アニメーションは300ms以内で完了すること
- 🟢 **NFR-003**: 初回ページロード時のレンダリングは1秒以内で完了すること

### アクセシビリティ要件
- 🟢 **REQ-401**: WCAG 2.1 AA準拠の実装
- 🟢 **REQ-402**: キーボードナビゲーションサポート
- 🟢 **REQ-403**: タッチ操作とマウス操作の両方をサポート
- 🟢 **NFR-201**: フッターメニューのタップ領域は44px以上
- 🟢 **NFR-202**: ハンバーガーメニューのタップ領域は44px以上

### 技術制約
- 🟢 **フレームワーク**: React 18+ with TypeScript、Mantine 7.x必須
- 🟢 **ブレークポイント**: 768px（Mantine 48em）固定
- 🟢 **状態管理**: React Context + useReducer + Mantine hooks
- 🟢 **ブラウザ対応**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+

- **参照したEARS要件**: NFR-001, NFR-002, NFR-003, REQ-401, REQ-402, REQ-403, NFR-201, NFR-202
- **参照した設計文書**: architecture.md「技術的制約と考慮事項」セクション

## 4. 想定される使用例（EARSEdgeケース・データフローベース）

### 基本的な使用パターン
- 🟢 **モバイル基本操作**: フッターメニューからメイン画面への遷移（REQ-103準拠）
- 🟢 **モバイル詳細操作**: ハンバーガーメニューから詳細画面への遷移（REQ-104準拠）
- 🟢 **デスクトップ基本操作**: ヘッダーメニューからメイン画面への遷移（REQ-105準拠）
- 🟢 **デスクトップ補助操作**: サイドメニューからその他画面への遷移（REQ-106準拠）

### レスポンシブ動作
- 🟢 **レイアウト自動切り替え**: 画面サイズ変更時のリアルタイム対応（REQ-301準拠）
- 🟢 **境界値動作**: 768px境界での適切なレイアウト切り替え（EDGE-101準拠）

### エラーケース・エッジケース
- 🟢 **EDGE-001**: JavaScript無効環境でも基本ナビゲーション機能維持
- 🟢 **EDGE-102**: 極小画面（320px）での適切な表示
- 🟢 **EDGE-103**: 極大画面（4K）での適切なスケーリング
- 🟡 **EDGE-202**: 画面回転時の適切なレイアウト再計算
- 🟡 **EDGE-203**: 複数タブ・ウィンドウ間でのサイズ変更処理

- **参照したEARS要件**: REQ-103, REQ-104, REQ-105, REQ-106, REQ-301, EDGE-001, EDGE-101, EDGE-102, EDGE-103, EDGE-202, EDGE-203
- **参照した設計文書**: architecture.md「レスポンシブ戦略」「レイアウト切り替え戦略」セクション
