# TASK-104: SideNavigation - テストケース定義

## 概要

SideNavigationコンポーネントの包括的なテストケースを定義し、TDDアプローチでの品質保証を実現します。

## テストケース

### 1. 基本表示テスト (FR-001)

#### 1.1 レスポンシブ表示テスト

**TC-104-001: デスクトップ表示テスト**
- **目的**: 1024px以上でのSideNavigation表示確認
- **前提条件**: ビューポート幅が1024px以上
- **実行手順**:
    1. SideNavigationコンポーネントをレンダリング
    2. nav要素の存在確認
    3. 初期表示状態の確認
- **期待結果**: SideNavigationが表示され、適切な幅（280px）で描画される
- **測定項目**: DOM要素の存在、スタイル属性の値

**TC-104-002: タブレット・モバイル非表示テスト**
- **目的**: 768px未満でのSideNavigation非表示確認
- **前提条件**: ビューポート幅が768px未満
- **実行手順**:
    1. 画面幅を768px未満に設定
    2. SideNavigationコンポーネントをレンダリング
    3. 表示状態の確認
- **期待結果**: SideNavigationが非表示（display: none）
- **測定項目**: CSSスタイルの適用状況

#### 1.2 ナビゲーション項目表示テスト

**TC-104-003: ナビゲーション項目レンダリングテスト**
- **目的**: 提供された項目が正しく表示されることの確認
- **前提条件**: NavigationItem配列が提供される
- **実行手順**:
    1. テスト用ナビゲーション項目を定義
    2. SideNavigationをレンダリング
    3. 各項目の表示確認
- **期待結果**: 全ての項目がアイコンとラベルで表示される
- **測定項目**: NavLink要素数、アイコン表示、テキスト内容

**TC-104-004: 階層構造表示テスト**
- **目的**: 2階層のナビゲーション構造の正確な表示
- **前提条件**: 親子関係を持つNavigationItem
- **実行手順**:
    1. 階層構造を持つテストデータを作成
    2. コンポーネントレンダリング
    3. 親子関係の表示確認
- **期待結果**: 親項目とサブメニューが適切に階層表示される
- **測定項目**: ネストした要素構造、インデント表示

### 2. インタラクション機能テスト (FR-002)

#### 2.1 折りたたみ機能テスト

**TC-104-005: トグル操作テスト**
- **目的**: クリックによる展開・折りたたみ動作の確認
- **前提条件**: SideNavigationが展開状態
- **実行手順**:
    1. トグルボタンをクリック
    2. 幅の変化確認（280px → 80px）
    3. 再度クリックで元に戻ることを確認
- **期待結果**: スムーズな幅変更とアニメーション
- **測定項目**: 幅の数値変化、アニメーション時間（300ms以内）

**TC-104-006: 状態保持テスト**
- **目的**: 折りたたみ状態のlocalStorage保存確認
- **前提条件**: ブラウザのlocalStorageが利用可能
- **実行手順**:
    1. SideNavigationを折りたたみ
    2. ページリロード
    3. 状態の復元確認
- **期待結果**: リロード後も折りたたみ状態が保持される
- **測定項目**: localStorage値、復元された表示状態

#### 2.2 キーボード操作テスト

**TC-104-007: キーボードナビゲーションテスト**
- **目的**: Tab、Enter、Escapeキーでの操作確認
- **前提条件**: SideNavigationにフォーカス
- **実行手順**:
    1. Tabキーで各項目へのフォーカス移動
    2. Enterキーでの項目選択
    3. Escapeキーでのフォーカス解除
- **期待結果**: 全操作が正常に動作する
- **測定項目**: フォーカス位置、選択状態、イベント発火

**TC-104-008: フォーカス管理テスト**
- **目的**: 適切なタブ順序とフォーカス表示の確認
- **前提条件**: 複数のナビゲーション項目
- **実行手順**:
    1. 最初の項目にフォーカス
    2. Tab操作での順次移動
    3. フォーカス表示の確認
- **期待結果**: 論理的なタブ順序とフォーカス表示
- **測定項目**: フォーカス順序、視覚的表示

### 3. アクセシビリティテスト (AR-001)

#### 3.1 ARIA属性テスト

**TC-104-009: ARIA属性設定テスト**

- **期待結果**: 全てのARIA属性が適切に設定される
- **測定項目**: role、aria-expanded、aria-current値

**TC-104-010: スクリーンリーダー対応テスト**
- **目的**: スクリーンリーダーでの読み上げ内容確認

    3. Live Regions設定確認
- **期待結果**: axe-core違反0件、適切な読み上げ情報
- **測定項目**: アクセシビリティ違反数、セマンティック構造

#### 3.2 コントラスト・操作性テスト

**TC-104-011: カラーコントラストテスト**
- **目的**: WCAG 2.1 AA基準のコントラスト比確認
- **前提条件**: ライト・ダークテーマ両方
- **実行手順**:
    1. 各テーマでのコントラスト比測定
    2. テキストと背景のコントラスト確認
    3. フォーカス状態のコントラスト確認
- **期待結果**: 全ての組み合わせで4.5:1以上
- **測定項目**: コントラスト比数値

**TC-104-012: タッチターゲットサイズテスト**
- **目的**: 44px以上のクリック領域確保確認
- **前提条件**: 各ナビゲーション項目
- **実行手順**:
    1. 各NavLinkのサイズ測定
    2. クリック可能領域の確認
    3. タッチデバイスでの操作確認
- **期待結果**: 全項目で44px以上のタッチターゲット
- **測定項目**: 要素のwidth、height値

#### 4.2 メモリ使用量テスト

**TC-104-015: メモリリークテスト**
- **目的**: 長時間使用でのメモリリーク検出
- **前提条件**: メモリプロファイル測定環境
