## 1. 正常系テストケース（基本的な動作）

### 1.1 プレイヤー一覧取得テスト

- **テスト名**: プレイヤー一覧取得_正常系
  - **何をテストするか**: 論理削除されていないプレイヤーのみが作成日時降順で取得されること
  - **期待される動作**: アクティブなプレイヤーのみがリストで返される
- **入力値**: なし（get_players()は引数なし）
  - **入力データの意味**: 既存の複数プレイヤーデータが前提
- **期待される結果**: `Vec<Player>`（is_deleted=0のプレイヤーのみ、created_at降順）
  - **期待結果の理由**: 削除されたプレイヤーを表示すべきではない
- **テストの目的**: 一覧表示機能の基本動作確認
  - **確認ポイント**: 論理削除フィルタリング、ソート順の正確性
- 🔵 **青信号**: 要件定義書のAPI仕様とLinear Issue詳細を参照

- **テストの目的**: 基本的な作成機能の確認
